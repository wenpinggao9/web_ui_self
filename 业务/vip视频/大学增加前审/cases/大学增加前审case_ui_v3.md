# 大学增加前审

## 前审功能

##### 测试项目
大学增加前审

##### 备注
- 大学前审遇到非目标题目时，全部选择「题目内容涉及不良导向/敏感信息」让工单退场，不占用后续 case 的题池资源
- 同角色跨用例复用浏览器会话（`config.yaml` 中 `cross_case_session: true`），除角色切换外不在每个 case 重复登录
- 建议执行顺序：C1→C2→C3→…→C10（主链）→ C11（K12，依赖 C2）→ C12→C13→C14（需主链题池释放）→ C15–C18 → C19–C24（独立/穿插）

### Case 1

#### 用例ID：vip_前审_001

##### 验证点
投放大学题目经过前审环节

##### 优先级
P0

##### 前置条件
- 投放策略来源、大学学段、数学学科的题目（2题），获取 orderId1 和 orderId2

##### 操作步骤
1. 管理员从全部题目中搜索来源：策略
2. 学段选择「大学」
3. 学科选择「数学」
4. 提交查看结果

##### 预期结果
- 查询到的列表中存在工单ID=${orderId1}和工单ID=${orderId2}
- 工单ID=${orderId1}和工单ID=${orderId2}的生产状态都为「待前审」
